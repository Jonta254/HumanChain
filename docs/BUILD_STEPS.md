# HumanChain Build Steps

This file remembers the path from idea to launch.

## Step 1: Create GitHub Repository

- Repository name: `HumanChain`
- Description: `A World Mini App where verified humans ask real questions, join daily global chains, read monthly human stories, and get the world's verdict.`
- Recommended visibility: Public
- Recommended starter choices: README, Node gitignore, optional MIT license

## Step 2: Create Mini App Foundation

- Use Next.js as the frontend app framework.
- Use `@worldcoin/minikit-js` for MiniKit commands.
- Use a mobile-first bottom tab layout.
- Keep all important actions within one tap from the homepage.

## Step 3: HumanChain Core Product

- Home
- Ask The World
- Human Verdict
- Daily Human Chain
- Monthly Human Story
- Human Streak
- Me/Profile

## Step 4: World Native Features

- Wallet Auth for sign-in.
- Verify command for one-human actions.
- Pay command for WLD features from 1 WLD to 6 WLD.
- Request permissions for notifications and microphone.
- Notifications for verdicts, streaks, answers, tips, and new stories.
- Share and share contacts for growth.

## Step 5: First MVP

- Build the UI shell.
- Wire MiniKit helper functions.
- Add backend routes for payment references and payment confirmation.
- Add database storage.
- Test inside World App.
- Submit to Mini App Store review.
